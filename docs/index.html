Hydroponic Automation System

The project "IoT-Based Automated Hydroponic System" aims to develop a modern and sustainable approach to hydroponic farming that maximizes efficiency by using technology to monitor and control growth conditions, reducing resource consumption, and enabling users to make data-driven decisions for better crop management.

The Technology We Use

System

Our solution includes a dedicated mobile application, empowering users with continuous access and control over the hydroponic system. This mobile platform ensures round-the-clock interaction and monitoring capabilities, offering users unparalleled convenience and oversight at any hour.

For the web application while logging in the normal user will get the same facilities as the mobile user. But for the admin user, he/she will be able to add the kind of plants we can grow in the hydroponics system and preferred conditions for the plants. The user management and the registering the new system will also handled in the web application

High-level system overview

Method used to ensure secure communication:

MQTT protects the user from “Man-in-the-Middle” attacks where hackers can use vulnerabilities in public networks to steal data transmitted to the viewer. Using TLS encryption to mask a user's connection with the website can prevent this sort of attack.

Flow Control

The system consists of four main endpoints:

  • Hardware in Farmer's end
  • Mobile application in Farmer's end
  • Web application in Administrator's end
  • Database and server application hosted on the Cloud

Security

We ensure security using 3 main methods

Security is a key concern in our product since we deal with highly sensitive data related to plants and personal security. We ensure security using 3 main methods:

  • Isolation
  • Encyrption
  • Authorization and authentication

Encryption in Server and Database

The secure MQTT protocol, TLS ensures secure data encryption both at transit and rest using TSL and other standards

Encrypting locally stored data

To ensure that farmers or any other third party cannot tamper locally stored recordings, they are encrypted using AES-256. Node.js's built-in cipher class File-encrypter is used to encrypt the recordings.AWS Key Management Service is used to control and manage the encryption keys.

Encryption in AWS server

AWS provides end-to-end and hop-to-hop encryption for IOT Devices. The Things are certificate protected and policy protected.
